2.Explain Django Architecture ?

Architecture Architecture 
    
                    Django follows the Model-View-Controller (MVC) architectural pattern, 
                    
                    but with a slight variation known as the Model-View-Template (MVT) pattern. 
                    
                    The MVT pattern is essentially a specialized version of MVC that is commonly used in Django web applications.
    
    
    
    
Model:

    
                 The Model represents the data layer of the application.
                 It defines the data structure, relationships, and business logic of the application's data.
                 Django provides an Object-Relational Mapping (ORM) layer that allows developers to define models as Python classes. 
                 Each model class represents a database table, and the attributes of the class define the table's fields. 
                 The Django ORM handles the interactions between the models and the database, making it easier to work with data.
    
    
    
    
View:

    
    
                The View handles the business logic and user interaction of the application.
                In Django, a View is a Python function or class that receives requests from the client and returns responses. 
                Views define how the application processes the requests, performs necessary operations using the models, and generates the appropriate response. 
                Views can render HTML templates, return JSON data, or redirect to other views.
    
    
    
    
Template:

    
    
                The Template handles the presentation layer of the application.
                It defines the structure and layout of the user interface. 
                Django uses its own template engine, which allows developers to create HTML templates with dynamic content. 
                Templates can contain placeholders and tags that are populated with data from the models.
                The template engine is responsible for rendering the templates into the final HTML that is sent to the client.
    
                In the MVT pattern, the Controller part of the MVC pattern is replaced by the View and the Template in Django. The View acts as the controller by handling the logic and data manipulation, while the Template takes care of the rendering and presentation aspects.

28.What is the response cycle in Django

        After that, Django will load a particular view, passing the HttpRequest as its first argument to the view function. Each view then returns an HttpResponse object.
    
        Firstly, the settings.py file, which contains various middleware classes, is loaded
    
        All the middleware classes get executed in the same order in which they are mentioned
    
        Now, the request will be moved to the URL Router. 
        
        The URL Router gets the URL path from the request and later tries to map with the given URL paths within the urls.py.
    
        After mapping, it calls the equivalent view function, from where the corresponding response is generated.
    
        The response now passes through the response middleware and is sent back to the client/browser
